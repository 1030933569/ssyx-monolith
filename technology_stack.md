# 本科毕业设计技术栈总结

本项目采用前后端分离的架构设计，系统划分为 **后端服务端**（包含管理端 API 和小程序端 API）和 **前端移动端**（原生微信小程序端）。以下是详细的技术框架总结列表，可供论文编写参考：

## 1. 后端技术栈 (Backend Stack)
后端主要基于 Java 语言开发，采用了目前企业界主流的 Spring Boot 单体微服务融合架构，技术成熟度高、社区生态完善。

*   **核心框架**：`Spring Boot (v2.3.6.RELEASE)`
    *   作用：作为核心开发脚手架，提供自动装配、快速启动并简化依赖版本管理能力。
*   **数据持久层框架**：`MyBatis-Plus (v3.4.1)`
    *   作用：作为 MyBatis 的功能增强工具包，简化单表基本增删改查操作，并内置高效的分页拦截器。
*   **关系型数据库**：`MySQL (v5.7)` + `MySQL Connector/J (v8.0.33)`
    *   作用：系统的核心底层数据存储（以 Docker 方式部署 5.7 版本运行），用于存储商城商品、订单交易记录及核心用户数据。
*   **非关系型数据库与内存计算**：`Redis (v6.0)`
    *   作用：构建高性能 Key-Value 缓存层，用于拦截大量频繁读取（如首页商品信息）、实现分布式锁防止高并发超卖；同时在此项目中充当轻量级**消息中间件**。。
*   **认证与鉴权**：`JWT (JSON Web Token, v0.7.0)`
    *   作用：生成包含用户身份识别信息的加密令牌，实现轻量级、无状态的前后端访问会话鉴权。

*   **底层运行环境**：`Docker` 与 `Docker Compose` 编排系统。

---

## 2. 小程序前端技术栈 (Frontend Stack)
针对“社区团购”场景极速加载体验要求、深度获客需要，本项目的终端业务选择不使用打包框架（如 UniApp、Taro 等），而是使用微信官方最贴近系统核心的原生范式构建。

*   **业务开发规范与框架层**：
    *   `原生 Native 微信小程序生命周期机制`（App.json 等全家桶配置）。
*   **视图层渲染语言**：
    *   **排版构建**：`WXML` (WeiXin Markup Language)，微信封装的一种标记性描述语言，利用其组件模版语法进行 UI 树的编排。
    *   **样式体系**：`WXSS` (WeiXin Style Sheets)，使用微信独家设计的 `rpx` (Responsive Pixel) 响应式像素单位，能够在各式各样的 iOS 和安卓屏幕材质中保持等比例高清缩放自适应适配。
*   **客户端脚本与动作**：
    *   `JavaScript (ECMAScript 6+)` 配合微信 Page 生命周期函数绑定，实现页面跳转、触控反应、业务渲染能力。
*   **网络通信重构**：
    *   由于小程序底层的 `wx.request` 对于庞杂的 HTTP 请求不好管理，底层中基于 **Promise 化** 设计了专有 `http.js` 网络请求中心，并封装有“发包拦截器”，用于自动附加用户的 Token 认证与自动拦截接口出错处理。

---

## 总体架构简析 (论文参考点)
1. 架构上在当前具有很好的折中性，不仅适合本篇幅轻量集中的系统部署，其组件式划分、基于 Redis 实现的数据边界控制，使得项目后期如果要拆分成真正的 Spring Cloud 微服务，可以十分柔和地平稳过渡。
2. 小程序端采用纯原生机制最大化规避了因为三方套壳导致的性能损耗、白屏时间及未知兼容 bug，使得如“添加购物车”、“多层嵌套滑动视图”时的动画达到了每秒 60 帧的流畅体验。
